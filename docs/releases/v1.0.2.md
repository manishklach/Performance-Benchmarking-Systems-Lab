# AMD APU Toolkit v1.0.2

## Highlights

- added a dedicated `Sensors` browser view
- added a sensor provider badge in the dashboard header
- added optional sensor-provider integration status to the UI
- added WPR/ETW trace capture controls to the browser dashboard
- added `trace-preflight` and `capture-trace` CLI commands
- added trace preflight diagnostics, decoded WPR permission hints, and trace metadata output

## Browser dashboard

- new `Sensors` view for:
  - CPU MHz
  - CPU perf %
  - GPU core clock when available
  - GPU temperature when available
  - GPU power when available
  - throttle hints
- `Overview` now includes:
  - ETW trace start/stop controls
  - trace status
  - trace preflight state
  - trace output path
  - trace summary path

## Tracing

- new `trace-preflight` command checks:
  - WPR availability
  - `tracerpt` availability
  - elevation state
  - trace readiness
- new `capture-trace` command captures short WPR traces using profiles such as:
  - `CPU`
  - `GPU`
  - `Video`
  - `DiskIO`
  - `Power`
- successful captures write ETL metadata next to the trace
- failed captures now surface clearer hints, including the common `0xc5585011` policy/elevation block

## Docs

- expanded README with:
  - trace capture workflow
  - elevated-shell instructions
  - CLI examples
  - troubleshooting for trace permissions and sensor providers
  - updated feature matrix

## Known limitation

- on the current test machine, WPR capture is blocked from a non-elevated shell, so trace preflight reports `not ready` until the shell is run as Administrator
